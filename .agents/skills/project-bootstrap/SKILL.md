# Project Bootstrap

## 目的

在任务正式进入需求分析、技术设计或实现之前，先统一完成上下文加载与复杂度判级。

## 必读文件

1. `AGENTS.md`
2. `project_info.md`
3. `docs/architecture/middleware_contract.md`（若需求涉及跨模块约定或中间件）

## 输出要求

必须输出以下内容：

1. 当前需求理解摘要
2. 影响模块与中间件初判
3. 建议复杂度等级 `L1/L2/L3`
4. 是否建议拆分 `一期 / 二期`
5. 判级与拆分依据
6. 本次预计产出的文档清单
7. 建议创建的模块目录与当前期次目录
8. 请求用户确认等级和期次拆分后再进入下一阶段

## 强制约束

- 未确认复杂度等级前，不允许进入正式需求文档编写
- 未确认是否拆分 `一期 / 二期` 前，不允许直接创建最终文档目录
- 若任务已绑定某个模块期次目录，继续前必须读取该目录 `feature_info.md`
