# 前后端联调 — 后端缺失接口清单

前端 `toLink-Web` 的 service 层已定义以下接口调用，但后端 Controller 中尚未实现。
本文档用于跟踪这些待补充的接口。

---

## 1. 更新数据集

| 项目 | 内容 |
|---|---|
| **前端调用** | `PATCH /api/v1/datasets/{datasetId}` |
| **前端位置** | `toLink-Web/src/services/dataset.ts` → `updateDataset()` |
| **请求体** | `UpdateDatasetRequest`：`name?: string`, `description?: string` |
| **响应** | `Result<Void>` |
| **建议实现位置** | `DatasetController` + `DatasetService` |
| **备注** | 前端已定义 `UpdateDatasetRequest` 类型，对应 Java 需新增同名 DTO |

---

## 2. 更新对话

| 项目 | 内容 |
|---|---|
| **前端调用** | `PATCH /api/v1/chat/conversations/{id}` |
| **前端位置** | `toLink-Web/src/services/chat.ts` → `updateConversation()` |
| **请求体** | `UpdateConversationRequest`：`title?: string`, `isPinned?: boolean` |
| **响应** | `Result<Void>` |
| **建议实现位置** | `ChatController` + `ChatService` |
| **备注** | 用于修改对话标题和置顶状态 |

---

## 3. 发送消息（对话核心功能）

| 项目 | 内容 |
|---|---|
| **前端调用** | `POST /api/v1/chat/conversations/{conversationId}/messages` |
| **前端位置** | `toLink-Web/src/services/chat.ts` → `sendMessage()` |
| **请求体** | `SendMessageRequest`：`content: string`, `configId?: number` |
| **响应** | `Result<MessageDTO>` |
| **建议实现位置** | `ChatController` + `ChatService` |
| **备注** | 这是对话功能的核心接口。需要：(1) 保存用户消息；(2) 调用 RAG 服务生成 AI 回复；(3) 保存 AI 回复；(4) 返回 AI 回复的 MessageDTO。目前项目已有 `SaveMessageRequest`（在 link-model 中）可作为参考 |

## 附注：SSE 认证问题

后端 SSE 端点 `GET /api/v1/datasets/{datasetId}/files/parse-events` 使用 `@SaCheckLogin`，
要求 `satoken` header。但浏览器原生 `EventSource` 不支持自定义 header。

前端目前未使用该 SSE 端点，待实现时需考虑：
- 方案 A：后端增加 `satoken` query 参数兜底
- 方案 B：前端使用支持自定义 header 的 SSE 库（如 `eventsource-parser`）
