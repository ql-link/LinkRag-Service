# 知识文件上传与解析模块需求分析

## 1. 需求概述

### 1.1 背景

ToLink 需要提供面向对话会话空间的知识文件上传能力。用户在某个对话中上传文件后，Java 端负责接收原始文件、校验文件、保存原文件、写入原文件表，并通知 Python/RAG 端异步解析文件。

文件处理链路分为两类数据：

- 原文件：用户上传的原始文件，由 Java 端负责保存和写入原文件表。
- 解析文件：Python/RAG 端解析原文件后生成的解析结果，由 Python/RAG 端负责保存和写入解析文件表。

本需求重点定义 Java 端文件上传入口、原文件记录、会话空间归属、解析任务通知，以及 Java 与 Python/RAG 端的职责边界。Python/RAG 端具体解析算法、切片、embedding、向量索引不在本文档展开。

### 1.2 目标

- 支持登录用户在指定对话会话空间中上传知识文件。
- Java 端负责校验文件格式、文件大小和用户权限。
- 支持管理员查看并修改知识文件上传配置。
- Java 端负责保存原文件并写入原文件表。
- 原文件表必须记录文件所属会话空间、上传状态、失败原因等信息。
- 原文件上传成功后，Java 端通过消息队列通知 Python/RAG 端进行解析。
- Python/RAG 端解析完成后，负责保存解析文件并写入解析文件表。
- 原文件表和解析文件表都需要有字段标识成功或失败状态。
- 用户可以查看 Java 端上传结果，但当前 Java 端不展示 Python/RAG 最终解析状态。
- 支持解析任务通知失败或解析流程失败后的用户重试。
- 支持用户删除文件记录。
- Java 调用 Python/RAG 端接口需要服务间鉴权。

### 1.3 成功标准

- 登录用户可以在指定会话空间上传支持格式文件。
- 未指定会话空间时，上传请求被拒绝。
- 文件格式不支持或文件大小超过管理员配置时，上传失败并返回明确原因。
- 管理员修改上传配置后，新上传请求按最新配置生效。
- 原文件上传成功后，Java 端对象存储中存在原文件，数据库原文件表存在对应记录。
- 原文件表记录上传成功或失败状态。
- 原文件上传成功后，Java 端可以成功通知 Python/RAG 端进行解析。
- Python/RAG 端解析后，将解析文件写入对象存储，并将解析结果记录写入解析文件表。
- 解析文件表记录解析成功或失败状态。
- 用户可以对失败任务发起重试。
- 用户可以删除自己有权限的文件记录。

## 2. 用户与场景

### 2.1 用户角色

| 角色 | 说明 | 权限边界 |
| --- | --- | --- |
| 普通用户 | 在对话会话空间中上传知识文件的用户 | 只能上传、查询、重试、删除自己有权限的会话空间文件 |
| 管理员 | 管理上传限制和排障的用户 | 可查看并修改知识文件上传配置，可按权限查看文件记录和失败原因 |
| Java 服务 | ToLink 后端服务 | 负责上传入口、校验、原文件存储、原文件表记录和解析任务通知 |
| Python/RAG 服务 | 外部解析服务 | 接收解析通知，解析原文件，保存解析文件并写入解析文件表 |

### 2.2 使用场景

| 场景 | 触发条件 | 用户目标 |
| --- | --- | --- |
| 上传知识文件 | 用户在某个对话中选择文件并上传 | 将文件加入当前会话空间的知识上下文 |
| 上传失败 | 文件格式不支持、大小超限、存储失败或鉴权失败 | 根据明确原因重新选择文件或稍后重试 |
| 触发解析 | 原文件上传成功 | 自动通知 Python/RAG 端异步解析 |
| 解析失败重试 | 通知失败或解析结果失败 | 用户或管理员重新触发解析 |
| 删除文件 | 用户不再需要某个会话空间文件 | 删除或标记删除文件记录，避免继续使用 |
| 管理配置 | 管理员调整知识文件上传配置 | 控制不同环境或业务阶段的上传限制 |

## 3. 范围定义

### 3.1 必须支持

- 支持登录用户单文件上传。
- 上传时必须指定文件归属的对话会话空间。
- 支持文件格式白名单：`.md`、`.markdown`、`.pdf`、`.docx`、`.txt`。
- 支持管理员配置单文件大小上限。
- 支持管理员查看当前知识文件上传配置。
- 支持管理员修改知识文件上传配置。
- 支持 Java 端保存原文件。
- 支持 Java 端写入原文件表。
- 原文件表记录会话空间 ID、用户 ID、原始文件名、文件大小、文件类型、存储引用、上传状态、失败原因等信息。
- 原文件上传成功后，Java 端通过消息队列通知 Python/RAG 端解析。
- Java 端通过外部接口触发 Python/RAG 端 MQ 投递能力，并进行服务间鉴权。
- 支持 Python/RAG 端解析后写入解析文件表。
- 解析文件表记录原文件关联、解析文件存储引用、解析状态、失败原因等信息。
- 支持上传或解析失败后的用户重试。
- 支持用户删除文件记录。
- 支持 Java 端展示上传状态。
- 当前 Java 端不展示 Python/RAG 最终解析状态。

### 3.2 可选增强

- 多文件批量上传。
- 上传进度展示。
- 文件 hash 去重。
- 管理员查看全量文件处理链路。
- Java 端展示 Python/RAG 最终解析状态。
- 删除文件时同步清理对象存储文件。
- 解析完成后通过回调或事件通知 Java 端。

### 3.3 暂不支持

- 未登录用户上传文件。
- 不绑定会话空间的文件上传。
- 除 `.md`、`.markdown`、`.pdf`、`.docx`、`.txt` 之外的文件格式。
- Java 端解析 PDF、Word、TXT 等文件内容。
- Java 端执行切片、embedding 或向量索引。
- Java 端同步等待 Python/RAG 解析完成。
- Java 端直接写入解析文件表。
- Java 端展示 Python/RAG 最终解析状态。
- 通过 MQ 消息体传输大文件内容。

## 5. 业务流程

### 5.1 原文件上传主流程

1. 用户在某个对话会话空间中选择文件。
2. 前端提交文件和会话空间标识。
3. Java 端校验用户登录态。
4. Java 端校验用户是否有权限向该会话空间上传文件。
5. Java 端校验文件是否为空。
6. Java 端校验文件后缀是否属于 `.md`、`.markdown`、`.pdf`、`.docx`、`.txt`。
7. Java 端读取管理员配置的单文件大小上限，并校验文件大小。
8. Java 端创建原文件记录，状态为上传处理中。
9. Java 端保存原文件到对象存储。
10. Java 端更新原文件记录，状态为上传成功，保存原文件存储引用。
11. Java 端通过外部接口触发 Python/RAG 端 MQ 投递能力，通知其解析该原文件。
12. Java 端记录解析通知是否提交成功。
13. Java 端向前端返回原文件 ID、上传状态和解析通知提交状态。

### 5.1A 管理员配置管理流程

1. 管理员进入知识文件配置管理页面。
2. 管理端查询当前知识文件上传配置。
3. 管理员查看当前单文件大小上限、允许格式白名单等配置项。
4. 管理员修改配置并提交保存。
5. Java 端校验管理员身份和配置内容合法性。
6. Java 端保存配置变更，并记录更新时间和操作人。
7. 配置保存成功后，新上传请求按最新配置执行。
8. 已经上传中的文件任务不要求回溯使用新配置。

### 5.2 Python/RAG 解析流程

1. Python/RAG 端收到解析任务消息。
2. Python/RAG 端根据消息中的文件引用下载原文件。
3. Python/RAG 端解析文件内容。
4. Python/RAG 端将解析后的文件或解析结果写入对象存储。
5. Python/RAG 端写入解析文件表。
6. 解析成功时，解析文件表记录解析成功状态。
7. 解析失败时，解析文件表记录解析失败状态和失败原因。

### 5.3 重试流程

1. 用户或管理员查看到文件上传或解析通知失败。
2. 用户点击重试。
3. Java 端校验用户登录态和文件权限。
4. 如果原文件上传失败，用户需要重新上传文件。
5. 如果原文件已上传成功但解析通知失败，Java 端重新通知 Python/RAG 端解析。
6. Java 端记录本次重试结果。

### 5.4 删除流程

1. 用户选择删除某个会话空间中的文件。
2. Java 端校验用户登录态和文件权限。
3. Java 端将原文件记录标记为删除或不可用。
4. 当前需求要求支持删除文件记录；是否同步删除对象存储文件作为后续增强。
5. 删除后，该文件不再出现在用户可用文件列表中。

### 5.5 异常流程

| 异常场景 | 期望行为 | 用户提示 |
| --- | --- | --- |
| 用户未登录 | 拒绝上传、查询、重试或删除 | 请先登录 |
| 未指定会话空间 | 拒绝上传 | 请选择文件所属对话 |
| 无会话空间权限 | 拒绝操作 | 无权操作该会话文件 |
| 文件为空 | 拒绝上传 | 请选择要上传的文件 |
| 文件格式不支持 | 拒绝上传 | 当前文件格式暂不支持 |
| 文件大小超过管理员配置 | 拒绝上传 | 文件大小超过限制 |
| 原文件保存失败 | 原文件表记录上传失败和失败原因 | 文件上传失败，请稍后重试 |
| 解析通知提交失败 | 原文件保留上传成功，记录解析通知失败 | 文件已上传，解析任务提交失败 |
| Python/RAG 解析失败 | 解析文件表记录解析失败和失败原因 | 当前 Java 端不展示最终解析状态 |
| 删除不存在或无权限文件 | 拒绝删除 | 文件不存在或无权删除 |

### 5.6 状态流转

| 对象 | 当前状态 | 触发动作 | 目标状态 | 说明 |
| --- | --- | --- | --- | --- |
| 原文件 | INIT | 创建原文件记录 | UPLOADING | 文件开始上传 |
| 原文件 | UPLOADING | 校验失败或存储失败 | UPLOAD_FAILED | 上传失败，记录原因 |
| 原文件 | UPLOADING | 原文件保存成功 | UPLOAD_SUCCESS | 原文件已保存 |
| 原文件 | UPLOAD_SUCCESS | 解析通知提交成功 | PARSE_NOTICE_SENT | 已通知 Python/RAG 解析 |
| 原文件 | UPLOAD_SUCCESS | 解析通知提交失败 | PARSE_NOTICE_FAILED | 可重试通知 |
| 原文件 | PARSE_NOTICE_FAILED | 用户重试成功 | PARSE_NOTICE_SENT | 重试通知成功 |
| 原文件 | 任意非删除状态 | 用户删除 | DELETED | 文件不可用 |
| 解析文件 | INIT | Python/RAG 创建解析记录 | PARSING | 解析中 |
| 解析文件 | PARSING | 解析成功 | PARSE_SUCCESS | 解析结果已保存 |
| 解析文件 | PARSING | 解析失败 | PARSE_FAILED | 记录失败原因 |

## 6. 功能需求

### 6.1 功能清单

| 功能点 | 描述 | 优先级 | 验收方式 |
| --- | --- | --- | --- |
| 配置查询 | 管理员查看当前知识文件上传配置 | P0 | 管理端可读取当前配置值 |
| 配置修改 | 管理员修改知识文件上传配置 | P0 | 修改成功后新上传请求按新配置执行 |
| 单文件上传 | 用户一次上传一个文件 | P0 | 支持 multipart/form-data 上传 |
| 会话空间绑定 | 上传时必须指定所属对话会话空间 | P0 | 缺少会话空间上传失败 |

| 格式校验 | 仅允许 `.md`、`.markdown`、`.pdf`、`.docx`、`.txt` | P0 | 上传其他格式失败 |
| 大小校验 | 单文件大小上限由管理员配置 | P0 | 超过配置上限上传失败 |
| 原文件存储 | Java 端保存用户上传的原始文件 | P0 | 对象存储存在原文件 |

| 状态记录 | 原文件表和解析文件表都记录成功或失败状态 | P0 | 数据库状态字段可查询 |
| 用户重试 | 支持失败后的用户重试 | P0 | 失败记录可重新触发 |
| 用户删除 | 支持用户删除自己的文件记录 | P0 | 删除后文件不可用 |
| 服务间鉴权 | Java 调 Python/RAG 接口需要鉴权 | P0 | 未携带有效凭证调用失败 |
| 上传状态展示 | Java 端展示上传成功或失败 | P0 | 前端可看到上传结果 |
| 解析状态展示 | 当前 Java 端不展示 Python/RAG 最终解析状态 | P0 | 前端不依赖最终解析状态 |

### 6.2 文件格式规则

- 首期支持格式：`.md`、`.markdown`、`.pdf`、`.docx`、`.txt`。
- 文件后缀标准化为小写。
- 后端必须校验文件格式。
- 文件格式校验不能只依赖前端。
- 不支持格式应直接拒绝上传，不创建有效原文件记录。

### 6.3 管理员配置规则

- 知识文件上传配置由管理员维护。
- 首期管理员可管理的配置项至少包括：
  - 单文件大小上限；
  - 允许上传的文件格式白名单。
- 管理端应支持查看当前生效配置。
- 管理端应支持修改并保存配置。
- 配置内容必须经过后端校验后方可生效。
- 配置变更后仅影响新发起的上传请求，不强制回溯已创建的上传或解析任务。
- 配置修改应具备操作人、操作时间等审计信息能力。

### 6.4 文件大小规则

- 单文件大小上限由管理员配置。
- 前端可以展示当前限制，但后端必须以配置值为准进行校验。
- 超过大小上限时，Java 端拒绝上传并返回明确错误。
- 管理员调整大小上限后，新上传请求按新配置执行。

### 6.5 文件格式配置规则

- 允许格式白名单由管理员配置。
- 前端可以展示当前允许格式，但后端必须以配置值为准进行校验。
- 管理员保存配置时，不允许出现空白后缀、重复后缀或非法格式项。
- 管理员收窄允许格式后，仅影响后续上传请求，不影响已上传文件记录。

### 6.6 原文件表规则

- 原文件表由 Java 端写入和维护。
- 原文件表必须记录文件所属会话空间。
- 原文件表必须记录上传成功或失败状态。
- 原文件表必须记录解析通知提交成功或失败状态。
- 原文件表必须记录失败原因。
- 原文件表必须记录对象存储中的原文件引用。

### 6.7 解析文件表规则

- 解析文件表由 Python/RAG 端写入和维护。
- 解析文件表必须关联原文件记录。
- 解析文件表必须记录解析成功或失败状态。
- 解析文件表必须记录解析文件存储引用。
- 解析失败时必须记录失败原因。
- 当前 Java 端不展示解析文件表中的最终解析状态。

### 6.8 Python/RAG 解析通知规则

- Java 端不直接集成 Python/RAG 侧 Kafka 或 RabbitMQ 客户端。
- Java 端通过外部 HTTP 接口触发 Python/RAG 侧 MQ 投递能力。
- 当前对接文档定义的触发端点为 `POST /api/v1/mq/send/parse-task`。
- 请求体核心字段包括 `task_id`、`document_id`、`file_url`、`file_type`。
- Python/RAG 侧消费目标为 `tolink.rag.parse_task`。
- Java 调用该接口必须进行服务间鉴权。
- 解析通知提交成功不代表解析完成。

### 6.9 权限规则

| 操作 | 文件所有者 | 管理员 | 未登录 |
| --- | --- | --- | --- |
| 上传文件 | 允许上传到自己有权限的会话空间 | 允许按权限上传 | 不允许 |
| 查询原文件记录 | 允许查询自己的会话空间文件 | 允许按权限查询 | 不允许 |
| 重试解析通知 | 允许重试自己的失败文件 | 允许按权限重试 | 不允许 |
| 删除文件记录 | 允许删除自己的文件 | 允许按权限删除 | 不允许 |
| 查看知识文件上传配置 | 不允许 | 允许 | 不允许 |
| 修改知识文件上传配置 | 不允许 | 允许 | 不允许 |

## 7. 数据需求

### 7.0 管理配置数据

| 数据项 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 配置项标识 | 字符串 | 是 | 如最大文件大小、允许格式白名单 |
| 配置值 | 字符串或数值 | 是 | 当前生效配置值 |
| 配置说明 | 字符串 | 否 | 管理端展示说明 |
| 最后修改人 | 字符串或数值 | 是 | 管理员标识 |
| 最后修改时间 | 时间 | 是 | 最近一次修改时间 |

说明：首期可通过配置中心、系统配置表或其他统一配置载体保存，具体落库方案在技术实现文档中确定。

### 7.1 原文件表需要保存的数据

| 数据项 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 原文件 ID | 字符串或数值 | 是 | Java 端原文件记录唯一标识 |
| 会话空间 ID | 字符串或数值 | 是 | 文件所属对话会话空间 |
| 用户 ID | 数值 | 是 | 上传用户 |
| 原始文件名 | 字符串 | 是 | 用户上传时的文件名 |
| 文件后缀 | 字符串 | 是 | 标准化为小写 |
| 文件大小 | 数值 | 是 | 原始文件字节数 |
| Content-Type | 字符串 | 否 | 上传请求中的内容类型 |
| 文件 Hash | 字符串 | 否 | 用于完整性校验或后续去重 |
| 原文件 object key | 字符串 | 是 | Java 端保存原文件的位置 |
| file_url | 字符串 | 是 | Python/RAG 端可读取的原文件引用 |
| 上传状态 | 枚举 | 是 | UPLOADING、UPLOAD_SUCCESS、UPLOAD_FAILED |
| 解析通知状态 | 枚举 | 是 | PARSE_NOTICE_PENDING、PARSE_NOTICE_SENT、PARSE_NOTICE_FAILED |
| 解析任务 ID | 字符串 | 是 | Java 通知 Python/RAG 时使用的 task_id |
| document_id | 字符串 | 是 | 传给 Python/RAG 的文档标识 |
| 失败原因 | 字符串 | 否 | 上传失败或解析通知失败原因 |
| 是否删除 | 布尔值或状态 | 是 | 标识记录是否已删除 |
| 创建时间 | 时间 | 是 | 记录创建时间 |
| 更新时间 | 时间 | 是 | 最近状态更新时间 |

### 7.2 解析文件表需要保存的数据

| 数据项 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 解析文件 ID | 字符串或数值 | 是 | Python/RAG 端解析文件记录唯一标识 |
| 原文件 ID | 字符串或数值 | 是 | 关联 Java 端原文件记录 |
| 会话空间 ID | 字符串或数值 | 是 | 冗余记录所属会话空间，便于查询 |
| 用户 ID | 数值 | 是 | 文件归属用户 |
| 解析文件 object key | 字符串 | 否 | 解析结果文件存储位置 |
| 解析状态 | 枚举 | 是 | PARSING、PARSE_SUCCESS、PARSE_FAILED |
| 失败原因 | 字符串 | 否 | 解析失败原因 |
| 创建时间 | 时间 | 是 | 解析记录创建时间 |
| 更新时间 | 时间 | 是 | 最近状态更新时间 |

### 7.3 数据约束

- 原文件必须绑定会话空间 ID。
- 原文件必须绑定上传用户 ID。
- 原文件上传成功时必须存在原文件 object key。
- 解析通知成功时必须存在 task_id 和 document_id。
- 解析文件必须关联原文件 ID。
- 上传失败或解析失败必须保存失败原因。
- 删除后的文件不应继续参与当前会话空间的可用知识文件列表。

### 7.4 数据留存与隐私

- 原文件和解析文件都属于用户数据，需要按用户和会话空间隔离访问。
- file_url、object key、task_id 不应无权限公开。
- 用户删除记录后，当前需求要求文件不可用；对象存储物理删除可作为后续增强。
- 服务间接口鉴权信息不得写入日志或返回给前端。

## 10. 验收标准

| 编号 | 验收项 | 验收方式 |
| --- | --- | --- |
| AC-1 | 登录用户可以在指定会话空间上传支持格式文件 | 上传 `.md`、`.markdown`、`.pdf`、`.docx`、`.txt` 文件，返回原文件 ID |
| AC-2 | 未登录用户不能上传 | 不带认证信息上传，返回认证失败 |
| AC-3 | 未指定会话空间不能上传 | 不传会话空间 ID，上传失败 |
| AC-4 | 无会话空间权限不能上传 | 使用无权限会话空间上传，返回无权限 |
| AC-5 | 不支持格式上传失败 | 上传非白名单后缀文件，返回格式不支持 |
| AC-6 | 文件大小超过管理员配置时上传失败 | 管理员配置上限后，上传超限文件失败 |
| AC-6A | 管理员可以查看当前知识文件上传配置 | 管理端查询返回当前大小上限和允许格式 |
| AC-6B | 管理员可以修改知识文件上传配置 | 管理员保存新配置后，新上传请求按新配置生效 |
| AC-7 | 原文件上传成功后写入原文件表 | 数据库存在原文件记录，上传状态为 UPLOAD_SUCCESS |
| AC-8 | 原文件上传失败时写入失败状态 | 模拟对象存储失败，原文件表记录 UPLOAD_FAILED 和失败原因 |
| AC-9 | 原文件上传成功后通知 Python/RAG 解析 | 成功调用 `POST /api/v1/mq/send/parse-task` 并记录通知状态 |
| AC-10 | 解析通知失败可重试 | 模拟 Python/RAG 接口失败后，用户可重新触发通知 |
| AC-11 | 用户可以删除自己的文件记录 | 删除后文件不再出现在可用列表 |
| AC-12 | Java 端当前不展示最终解析状态 | 前端列表只展示上传状态和解析通知状态 |
| AC-13 | Python/RAG 端解析后写入解析文件表 | Python/RAG 端解析成功或失败均有解析状态记录 |
| AC-14 | Java 调 Python/RAG 接口需要鉴权 | 缺少或错误鉴权信息时调用失败 |

## 11. 假设与依赖

### 11.1 假设

- 首期以单文件上传为主。
- 文件必须绑定到一个对话会话空间。
- 知识文件上传配置由管理员维护，首期至少包括单文件大小上限和允许格式白名单。
- Java 端负责原文件存储和原文件表记录。
- Python/RAG 端负责解析文件存储和解析文件表记录。
- Java 端通过外部 HTTP 接口触发 Python/RAG 侧 MQ 投递能力。
- Java 端当前不展示 Python/RAG 最终解析状态。
- 用户删除文件记录后，文件不再参与会话空间知识上下文。

### 11.2 外部依赖

- 对象存储服务及 bucket/prefix 规划。
- Python/RAG 侧 MQ 中台接口：`POST /api/v1/mq/send/parse-task`。
- Python/RAG 侧解析任务消费目标：`tolink.rag.parse_task`。
- 用户认证与会话空间权限体系。
- 管理员配置中心或配置表，用于控制知识文件上传配置。
- 前端文件上传、上传结果展示、重试和删除能力。

### 11.3 已有参考资料

- `docs/需求与技术文档/文件上传模块/LLM端与Java端对接.md`：定义 Java 端触发 Python/RAG MQ 中台 HTTP 接口方式。
- `toLink-components-oss` 组件：可作为对象存储抽象参考。
- 当前需求文档只定义需求边界，具体 API 路径、表结构、Service 设计应在技术实现文档中确定。
