# 文件上传模块数据模型

## 1. 当前模型说明

当前实现已经切换到“数据集中心化”模型：

- `dataset` 是文件和对话的共同归属实体
- `chat_conversation` 必须绑定 `dataset_id`
- `document_original_file` 直接归属于 `dataset_id`
- 解析任务通过 MQ 发放，不再建设 `document_parse_task` 表
- 删除语义为物理删除，不使用 `is_deleted`

## 2. 数据集表

```sql
CREATE TABLE `dataset` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '数据集ID',
  `user_id` BIGINT UNSIGNED NOT NULL COMMENT '所属用户ID',
  `name` VARCHAR(128) NOT NULL COMMENT '数据集名称',
  `description` VARCHAR(512) DEFAULT NULL COMMENT '数据集描述',
  `status` VARCHAR(32) NOT NULL DEFAULT 'ACTIVE' COMMENT '状态',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_dataset_user_name` (`user_id`, `name`),
  KEY `idx_dataset_user_updated` (`user_id`, `updated_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='数据集表';
```

## 3. 对话表

```sql
CREATE TABLE `chat_conversation` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '对话ID',
  `user_id` BIGINT UNSIGNED NOT NULL COMMENT '所属用户ID',
  `dataset_id` BIGINT UNSIGNED NOT NULL COMMENT '所属数据集ID',
  `last_config_id` BIGINT UNSIGNED DEFAULT NULL COMMENT '最后一次使用的模型配置ID',
  `last_model_name` VARCHAR(128) DEFAULT NULL COMMENT '最后一次使用的模型名称',
  `title` VARCHAR(255) NOT NULL COMMENT '对话标题',
  `is_pinned` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '是否置顶',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  KEY `idx_chat_conversation_user_updated` (`user_id`, `updated_at`),
  KEY `idx_chat_conversation_dataset_updated` (`dataset_id`, `updated_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='对话表';
```

## 4. 原始文件表

```sql
CREATE TABLE `document_original_file` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '原始文件ID',
  `dataset_id` BIGINT UNSIGNED NOT NULL COMMENT '所属数据集ID',
  `user_id` BIGINT UNSIGNED NOT NULL COMMENT '上传用户ID',
  `original_filename` VARCHAR(255) NOT NULL COMMENT '原始文件名',
  `file_suffix` VARCHAR(32) NOT NULL COMMENT '文件后缀',
  `file_size` BIGINT UNSIGNED NOT NULL COMMENT '文件大小(字节)',
  `content_type` VARCHAR(128) DEFAULT NULL COMMENT '内容类型',
  `bucket_name` VARCHAR(128) DEFAULT NULL COMMENT '存储桶名称',
  `object_key` VARCHAR(512) DEFAULT NULL COMMENT '对象Key',
  `file_url` VARCHAR(1024) DEFAULT NULL COMMENT '内部下载URL',
  `upload_status` VARCHAR(20) NOT NULL DEFAULT 'uploading' COMMENT '上传状态',
  `is_upload_success` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '上传是否成功',
  `parse_notice_status` VARCHAR(24) NOT NULL DEFAULT 'pending' COMMENT '解析通知状态',
  `parse_task_id` VARCHAR(36) NOT NULL COMMENT '解析任务业务ID',
  `parse_status` VARCHAR(20) NOT NULL DEFAULT 'not_started' COMMENT '解析状态',
  `is_parse_success` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '解析是否成功',
  `parsed_bucket_name` VARCHAR(128) DEFAULT NULL COMMENT '解析结果桶名',
  `parsed_object_key` VARCHAR(512) DEFAULT NULL COMMENT '解析结果对象Key',
  `parsed_file_url` VARCHAR(1024) DEFAULT NULL COMMENT '解析结果访问地址',
  `parsed_at` DATETIME DEFAULT NULL COMMENT '解析完成时间',
  `parse_failure_reason` VARCHAR(512) DEFAULT NULL COMMENT '解析失败原因',
  `failure_reason` VARCHAR(512) DEFAULT NULL COMMENT '上传或通知失败原因',
  `parse_notice_retry_count` INT NOT NULL DEFAULT 0 COMMENT '解析通知重试次数',
  `last_parse_notice_at` DATETIME DEFAULT NULL COMMENT '最近一次解析通知时间',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_parse_task_id` (`parse_task_id`),
  UNIQUE KEY `uk_dataset_original_filename` (`dataset_id`, `original_filename`),
  KEY `idx_document_dataset_created` (`dataset_id`, `created_at`),
  KEY `idx_document_user_created` (`user_id`, `created_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='原始文件表';
```

## 5. 存储路径规则

原始文件对象完整路径约定为：

```text
rag-raw/{userId}/{datasetId}/{yyyy}/{MM}/{dd}/{originalFilename}
```

其中：

- `rag-raw` 为业务语义前缀
- `userId` 为上传用户
- `datasetId` 为文件所属数据集
- 文件名必须使用原始文件名
- 如同一数据集下重名，上传请求直接失败，不做自动改名
