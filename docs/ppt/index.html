KnowledgeFileServiceImpl.upload()

上传流程：8 步详解

①assertOwnedDataset(userId, datasetId) ②validateFile(file) — Redis读运行时配置 ③assertNoDuplicateOriginalFilename() — COUNT(*) ④INSERT document_original_file (status=uploading)
⑤ossService.upload2PreviewUrl(PRIVATE, file, objectKey) ⑥UPDATE status=success, 设置 file_url ⑦initializeParseFileIfAbsent() — UPSERT parse_file ⑧TransactionSynchronization.afterCommit() → 自动解析
为什么 afterCommit？
保证 DB 已落盘再发 MQ，避免消费者读到不存在的记录

WHAT IF

如果出问题了会怎样

01 · 用户刷新页面
SSE 断了 → 前端回查 DB → 立刻拿到当前状态
02 · parse_result 消息丢了
卡住扫描 60s 后发现 → 重读 DB → 补推 SSE
03 · 旧任务结果晚到
当前任务过滤拦截 → 不推 SSE → 记审计日志
04 · Redis 抖动
读降级走 DB / 写失败回滚 + binlog 补偿删
